\doxysubsection{Detailed Description}
Unity\textquotesingle{}s Behavior-\/\+Driven Development (BDD) test feature. It allows developers to structure and describe various phases (Given, When, Then) of a test scenario in a BDD-\/style format.\hypertarget{readme.md_autotoc_md349}{}\doxysubsection{\texorpdfstring{Introduction}{Introduction}}\label{readme.md_autotoc_md349}
This project is based on the Unity framework originally created by Mike Karlesky, Mark Vander\+Voord, and Greg Williams in 2007. The project extends Unity by providing macros to define BDD structures with descriptive elements. Feature added by Michael Gene Brockus (Dreamer).\hypertarget{readme.md_autotoc_md350}{}\doxysubsection{\texorpdfstring{License}{License}}\label{readme.md_autotoc_md350}

\hypertarget{readme.md_autotoc_md352}{}\doxysubsubsection{\texorpdfstring{BDD Macros}{BDD Macros}}\label{readme.md_autotoc_md352}
The provided BDD macros allow you to structure your test scenarios in a descriptive manner. These macros are for descriptive purposes only and do not have functional behavior.
